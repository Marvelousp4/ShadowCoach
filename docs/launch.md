# Open-Source Launch Plan

## Positioning

Use one promise consistently:

> Import real English videos, repeat from memory, and see exactly which words, sounds, and rhythms need work.

Lead with the learner outcome. WhisperX, Azure, Praat, and LLM provider names belong in the technical section, not the headline.

## Launch Assets

- A 20–30 second silent-captioned GIF: import -> hidden listen -> record -> colored diagnosis.
- One clean macOS screenshot of the Library and Practice workflow.
- One feedback screenshot showing a word heatmap and phoneme detail.
- One iPhone screenshot showing portable practice.
- A signed and notarized DMG, checksum, and tested clean-install path.
- Five to ten small `good first issue` tasks ready before launch day.

Use original or public-domain practice material in every public asset.

## Release Sequence

1. Publish the repository and `v0.1.0` release together.
2. Post a concise demo to Show HN, Product Hunt, V2EX, Reddit language-learning communities, and macOS communities.
3. Publish a Chinese build story on 少数派、知乎、即刻、Bilibili and 掘金.
4. Post short daily before/after practice clips on 抖音 and 小红书 for two weeks.
5. Turn repeated questions into issues, setup docs, and short videos.

## Demo Script

```text
I built a Mac app because normal shadowing tools only tell me to repeat.
Here I import a real interview, listen with the text hidden, and repeat it.
The app finds the two words I missed, a weak final sound, and a broken pause.
Everything is saved locally, and the project is open source.
```

## Success Metrics

- Clean DMG install success rate.
- README visitor-to-download conversion.
- First successful listen/record/analyze session.
- Issues with reproducible reports rather than setup confusion.
- External contributors who successfully build within 15 minutes.

Stars are useful distribution, not the product goal. Optimize for successful practice sessions and contributors who return.
